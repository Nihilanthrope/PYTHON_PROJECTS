## 🐍 The Origin Story

> *Every great engineer has a repo they look back at and laugh. This is mine.*

This is where it started. **First year. No idea what I was doing. Python installed, tutorials half-watched, and way too much confidence.**

These aren't polished projects. They're **honest ones** — the kind where the error messages taught more than the tutorials did. Dice games, restaurant menus, life simulators, and one file that's literally just 20,000 lines of a string being printed a billion times.

### 🎲 PROJECT.py — The First Game

<div align="center">
  <img src="https://media.giphy.com/media/3oKIPnAiaMCws8nOsE/giphy.gif" width="220" alt="dice anime"/>

> The one that started it all. Roll two dice. The game ends when you say no — and it does **not** end gracefully.

```python
import random

while True:
    choise = input("Roll The Dice? (y/n): ")   # typo preserved for authenticity
    if choise == "y" or choise == "Y":
        die1 = random.randint(1, 6)
        die2 = random.randint(1, 6)
        print(f'({die1}, {die2})')
    elif choise == "n" or choise == "N":
        print("Maa Chuda Bsdk")   # peak first-year energy
        break
    else:
        print("invalid Hai lawde")

# my first project ig lol
```

**Concepts learned:** `random` module, `while True` loops, `if/elif/else`, `f-strings`, `break`

**What I'd do differently now:** Validate input with a function, add a score tracker, maybe not the exit message.

### 🍜 MENU.py — Restaurant Order System

<div align="center">
  <img src="https://media.giphy.com/media/BzyTuYCmvSORqs1ABM/giphy.gif" width="220" alt="food anime gif"/>
</div>

> A fully functional CLI restaurant order system. You can order pizza, momos, thupka, chai — it tracks your running total and handles bad input.

```python
menu = {
    "pizza": 180,     "burger": 60,
    "chowmin": 70,    "fried_chilli_momos": 70,
    "momos": 70,      "thupka": 60,
    "pasta": 60,      "coffee": 30,
    "juice": 70,      "chai": 20,
}

total = 0
print("======MENU==============")
for item, price in menu.items():
    print(f"{item.title():100} ₹{price}")
print("=========================")

while True:
    user = input("Your Order (X To Exit Menu) :- ").lower()
    if user == "x":
        print(f"Thanks For Visiting Us Your Total Is {total}")
        break
    if user in menu:
        total += menu[user]
        print(f"The Price Of {user} Will Be ₹{menu[user]}")
        print(f"Your Total Is {total}")
    else:
        print(f"Sorry, {user} Is Not Available Right Now")
```

**Concepts learned:** `dict`, `for item, value in .items()`, running totals, input validation, formatted strings with alignment

**Genuinely solid logic for a first-year project.** This is a working POS terminal in 30 lines.

### 💀 20KWRDS.py — The Chaos Experiment

> 20,878 lines. One mission: print a string one billion times per `print()` call. Called a hundred times per page. This is what happens when you discover string multiplication and have no supervision.

```python
a = "Acha Lawde"
b = "imnotdrunkyawr"
txt = " Ne Gand Maar di Apki "

print((b + txt) * 1000000000)   # × 20,000 more lines
```

**Concepts "learned":** String multiplication (`str * n`), the concept that your CPU is not invincible

## 🪞 Looking Back

<div align="center">
  <img src="https://media.giphy.com/media/3NtY188QaxDdC/giphy.gif" width="260" alt="thinking anime"/>
</div>

<br/>

This repo was written when I had no idea what I was doing. Here's what I'd tell that version of me now:

```python
# Then vs Now

# THEN
while True:
    choise = input("Roll The Dice? (y/n): ")   # typo, no validation, no function

# NOW
def roll_dice() -> tuple[int, int]:
    """Roll two six-sided dice. Returns a tuple of results."""
    return random.randint(1, 6), random.randint(1, 6)

def get_user_choice(prompt: str, valid: set) -> str:
    """Validated input loop with clean error handling."""
    while True:
        choice = input(prompt).strip().lower()
        if choice in valid:
            return choice
        print(f"  ✗ Invalid input. Choose from: {valid}")
```

**What this repo actually represents:**

- ✅ I wrote actual working programs from day one — not just `print("hello")`
- ✅ I understood loops, dictionaries, and branching without being taught properly
- ✅ `YOUR_LIFE_EXE.py` is a legitimate text game with ANSI formatting and deep branching
- ✅ `MENU.py` is production-viable logic for a small ordering system
- ✅ I was building things, not just following tutorials

The code is rough. The naming conventions are non-existent. The comments are chaotic. But there's **real problem-solving instinct** in here — and that's what actually matters at year one.
